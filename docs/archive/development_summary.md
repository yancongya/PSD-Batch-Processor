## 架构设计

### 分层架构

```
┌─────────────────────────────────────┐
│         用户界面层 (UI)              │
│      app/ui/main_window.py          │
├─────────────────────────────────────┤
│         业务逻辑层 (Core)            │
│  processor.py  |  photoshop_controller.py
├─────────────────────────────────────┤
│         数据模型层 (Models)          │
│            file_item.py             │
├─────────────────────────────────────┤
│         配置管理层 (Config)          │
│             settings.py             │
├─────────────────────────────────────┤
│         工具层 (Utils)               │
│              logger.py              │
└─────────────────────────────────────┘
```

### 关键设计模式

1. **单例模式**：Settings、Logger
2. **观察者模式**：回调函数（on_progress、on_status_update）
3. **策略模式**：单线程/多线程处理
4. **工厂模式**：FileItem 创建

   - 暗色/亮色主题

   - 实时进度反馈

   - 配置持久化

5. **可维护性强**
   - 模块化设计
   - 单一职责原则
   - 详细的文档

### ⚠️ 注意事项

1. **平台限制**：仅支持 Windows（依赖 COM）
2. **Photoshop 依赖**：需要安装 Photoshop
3. **并发限制**：建议单线程，避免内存问题
4. **脚本编写**：需要了解 JSX 语法
5. **首次使用**：需要手动配置 Photoshop 路径
