# PyQt-Fluent-Widgets V2 界面指南

本文档详细介绍 PyQt-Fluent-Widgets V2 版本的四页布局设计和各页面功能。

## 📑 页面概览

V2 版本采用**四页分离设计**，每个页面专注于特定功能：

| 页面 | 图标 | 功能 | 位置 |
|------|------|------|------|
| **文件处理** | 🏠 | 文件操作和批量处理 | 第1页（默认） |
| **脚本管理** | 📝 | 浏览和管理脚本 | 第2页 |
| **设置** | ⚙️ | 配置应用程序 | 第3页 |
| **日志** | 📋 | 查看处理日志 | 第4页 |

---

## 🏠 第1页：文件处理 (ProcessInterface)

**功能**: 核心功能页面，用于添加文件、选择脚本、开始批量处理

### 页面布局

```
┌─────────────────────────────────────────────────────────┐
│ 📁 文件处理                                               │
├─────────────────────────────────────────────────────────┤
│ ⚙️ 当前配置                                               │
│   Photoshop: C:\Program Files\Adobe\Photoshop\...       │
│   当前脚本: 3个脚本可用                                   │
├─────────────────────────────────────────────────────────┤
│ 📂 文件操作                                               │
│   [添加文件] [添加文件夹] [清空列表]                      │
│   ┌─────────────────────────────────────────────────┐   │
│   │ 文件名          │ 状态    │ 路径                │   │
│   ├─────────────────────────────────────────────────┤   │
│   │ image1.psd      │ 待处理  │ C:\...\image1.psd   │   │
│   │ image2.psd      │ 待处理  │ C:\...\image2.psd   │   │
│   └─────────────────────────────────────────────────┘   │
│   就绪 - 2 个文件                                        │
├─────────────────────────────────────────────────────────┤
│ ▶️ 处理控制                                               │
│   选择脚本: [脚本下拉框] [刷新]                          │
│   并发数: [1]                                            │
│   [开始处理] [停止] [打开输出文件夹]                     │
├─────────────────────────────────────────────────────────┤
│ 📊 处理进度 (开始处理后显示)                              │
│   [进度条]                                               │
│   处理中... - 1/2 (50%)                                  │
└─────────────────────────────────────────────────────────┘
```

### 核心控件

#### 1. 当前配置卡片
- **Photoshop 路径显示**: 显示当前配置的 Photoshop.exe 路径
- **当前脚本显示**: 显示可用脚本数量

#### 2. 文件操作卡片
- **添加文件按钮**: 打开文件选择对话框，支持多选 PSD 文件
- **添加文件夹按钮**: 选择文件夹，自动扫描所有 PSD 文件（包括子目录）
- **清空列表按钮**: 清空所有已添加的文件（需要确认）
- **文件列表 TreeWidget**:
  - 列1: 文件名
  - 列2: 状态（待处理/处理中/已完成/失败）
  - 列3: 完整路径
- **统计信息**: 显示当前文件数量

#### 3. 处理控制卡片
- **脚本选择下拉框**: 显示所有可用脚本（支持子目录）
- **刷新按钮**: 重新扫描脚本目录
- **并发数设置**: 1-8 可调，默认 1
- **开始处理按钮**: 开始批量处理（禁用状态直到有文件和脚本）
- **停止按钮**: 停止当前处理
- **打开输出文件夹**: 打开备份目录

#### 4. 进度卡片（动态显示）
- **进度条**: 显示处理进度百分比
- **进度信息**: 显示当前状态和进度（如"处理中... - 1/2 (50%)"）

### 使用流程

1. **设置 Photoshop 路径**（首次使用）
   - 转到"设置"页面
   - 选择 Photoshop.exe 路径
   - 点击"保存设置"

2. **添加文件**
   - 点击"添加文件"选择单个或多个 PSD 文件
   - 或点击"添加文件夹"批量添加文件夹内所有 PSD

3. **选择脚本**
   - 在"选择脚本"下拉框中选择要执行的 JSX 脚本
   - 如需刷新脚本列表，点击"刷新"按钮

4. **配置并发数**
   - 根据电脑性能调整并发数（建议 2-4）

5. **开始处理**
   - 点击"开始处理"按钮
   - 进度卡片会自动显示
   - 实时查看处理进度和日志

6. **查看结果**
   - 处理完成后会显示提示信息
   - 可在"日志"页面查看详细日志
   - 点击"打开输出文件夹"查看备份文件

## 📝 第2页：脚本管理 (ScriptInterface)

**功能**: 专门用于浏览、管理和查看脚本信息

### 页面布局

```
┌─────────────────────────────────────────────────────────┐
│ 📝 脚本管理                                               │
├─────────────────────────────────────────────────────────┤
│ 📂 脚本目录                                               │
│   当前目录: F:\...\scripts                              │
│   [刷新脚本列表] [打开脚本目录]                          │
├─────────────────────────────────────────────────────────┤
│ 📋 脚本列表                                               │
│   ┌─────────────────────────────────────────────────┐   │
│   │ 脚本名称          │ 类型    │ 路径              │   │
│   ├─────────────────────────────────────────────────┤   │
│   │ PsDeepCleaner.jsx │ 根目录  │ F:\...\scripts\.. │   │
│   │ optimize.jsx      │ examples│ F:\...\scripts\.. │   │
│   └─────────────────────────────────────────────────┘   │
│   找到 5 个脚本                                          │
├─────────────────────────────────────────────────────────┤
│ ℹ️ 脚本说明                                               │
│   • 脚本会自动扫描 scripts/ 目录及其子目录              │
│   • 支持 .jsx 格式的 Photoshop 脚本                     │
│   • 脚本会自动处理确认对话框，实现无人值守运行          │
│   • 在文件处理页面选择要执行的脚本                      │
└─────────────────────────────────────────────────────────┘
```

### 核心控件

#### 1. 脚本目录卡片
- **当前目录显示**: 显示脚本目录的完整路径
- **刷新脚本列表按钮**: 重新扫描脚本目录
- **打开脚本目录按钮**: 在文件资源管理器中打开脚本目录

#### 2. 脚本列表卡片
- **脚本列表 TreeWidget**:
  - 列1: 脚本名称（显示相对路径）
  - 列2: 类型（根目录/子目录名）
  - 列3: 完整路径
- **统计信息**: 显示找到的脚本总数

#### 3. 脚本说明卡片
- 使用说明和注意事项

### 脚本扫描特性

- **递归扫描**: 自动扫描 scripts/ 目录及其所有子目录
- **相对路径显示**:
  - 根目录脚本: 显示为 `脚本名.jsx`
  - 子目录脚本: 显示为 `子目录/脚本名.jsx`
- **类型分类**: 显示脚本所在子目录名称

### 使用场景

1. **查看可用脚本**
   - 浏览所有脚本及其路径
   - 了解脚本的组织结构

2. **查找特定脚本**
   - 通过 TreeWidget 查看完整路径
   - 确认脚本位置和类型

3. **刷新脚本列表**
   - 添加新脚本后刷新
   - 更新脚本目录结构

## ⚙️ 第3页：设置 (SettingsInterface)

**功能**: 集中管理应用程序的所有配置选项

### 页面布局

```
┌─────────────────────────────────────────────────────────┐
│ ⚙️ 设置                                                   │
├─────────────────────────────────────────────────────────┤
│ 🎨 Photoshop 设置                                        │
│   Photoshop 路径: [输入框] [浏览...]                     │
│   最大并发数: [4]                                       │
├─────────────────────────────────────────────────────────┤
│ 📂 路径设置                                               │
│   脚本目录: [输入框] [浏览...]                           │
│   备份目录: [输入框] [浏览...]                           │
├─────────────────────────────────────────────────────────┤
│ 🎨 主题设置                                               │
│   外观主题: [深色 ▼]                                    │
├─────────────────────────────────────────────────────────┤
│ [保存设置]                                               │
└─────────────────────────────────────────────────────────┘
```

### 核心控件

#### 1. Photoshop 设置卡片
- **Photoshop 路径输入框**: 显示和编辑 Photoshop.exe 路径
- **浏览按钮**: 打开文件选择对话框，定位 Photoshop.exe
- **最大并发数设置**: 1-8 可调，影响处理速度

#### 2. 路径设置卡片
- **脚本目录输入框**: 指定 JSX 脚本所在目录
- **备份目录输入框**: 指定备份文件保存目录
- **浏览按钮**: 选择目录路径

#### 3. 主题设置卡片
- **主题选择下拉框**:
  - 深色（Dark）
  - 浅色（Light）
- **实时切换**: 选择后立即生效

#### 4. 保存按钮
- **保存设置按钮**: 将所有配置保存到配置文件
- 显示保存成功/失败提示

### 配置项说明

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| Photoshop 路径 | Photoshop.exe 的完整路径 | 空（需手动设置） |
| 脚本目录 | JSX 脚本扫描目录 | `./scripts` |
| 备份目录 | 备份文件保存目录 | `./backups` |
| 最大并发数 | 同时处理的文件数量 | 1 |
| 主题 | 界面主题 | 深色 |

### 使用流程

1. **设置 Photoshop 路径**（必需）
   - 点击"浏览..."按钮
   - 定位到 Photoshop.exe（通常在 `C:\Program Files\Adobe\Photoshop\Photoshop.exe`）
   - 路径会自动填入输入框

2. **配置路径**（可选）
   - 如需修改脚本目录，输入新路径或点击"浏览..."
   - 如需修改备份目录，输入新路径或点击"浏览..."

3. **调整并发数**
   - 根据电脑性能调整（建议 2-4）
   - 高性能电脑可设置更高值

4. **切换主题**
   - 选择"深色"或"浅色"
   - 界面会立即切换主题

5. **保存配置**
   - 点击"保存设置"按钮
   - 配置会持久化保存，下次启动自动加载

## 📋 第4页：日志 (LogInterface)

**功能**: 查看、管理和保存处理日志

### 页面布局

```
┌─────────────────────────────────────────────────────────┐
│ 📋 处理日志                                               │
├─────────────────────────────────────────────────────────┤
│ [清空日志] [保存日志]                                    │
├─────────────────────────────────────────────────────────┤
│ ┌─────────────────────────────────────────────────────┐ │
│ │ [2026-01-24 14:30:15] [INFO] 脚本目录: F:\...\scripts│ │
│ │ [2026-01-24 14:30:15] [INFO] 备份目录: F:\...\backups│ │
│ │ [2026-01-24 14:30:20] [INFO] 开始处理 2 个文件      │ │
│ │ [2026-01-24 14:30:20] [INFO] 使用脚本: PsDeepCleaner│ │
│ │ [2026-01-24 14:30:20] [INFO] 并发数: 1              │ │
│ │ [2026-01-24 14:30:25] [INFO] image1.psd: 处理完成    │ │
│ │ [2026-01-24 14:30:30] [INFO] image2.psd: 处理完成    │ │
│ │ [2026-01-24 14:30:30] [SUCCESS] 处理完成: 2/2       │ │
│ └─────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

### 核心控件

#### 1. 控制按钮
- **清空日志按钮**: 清空当前显示的所有日志
- **保存日志按钮**: 将日志保存到文本文件

#### 2. 日志文本区域
- **只读文本框**: 显示所有日志内容
- **自动滚动**: 新日志自动滚动到底部
- **等宽字体**: 使用 Consolas/Monospace 字体，便于对齐
- **深色背景**: 便于长时间查看

### 日志格式

每行日志包含三个部分：
```
[时间戳] [级别] 消息内容
```

**日志级别**:
- `[INFO]`: 一般信息
- `[WARNING]`: 警告信息
- `[ERROR]`: 错误信息
- `[SUCCESS]`: 成功信息

**示例**:
```
[2026-01-24 14:30:15] [INFO] 脚本目录: F:\插件脚本开发\PSD Batch Processor\scripts
[2026-01-24 14:30:25] [INFO] image1.psd: 处理完成
[2026-01-24 14:30:30] [ERROR] Photoshop 未启动
```

### 使用场景

1. **查看处理进度**
   - 实时查看文件处理状态
   - 监控批量处理进度

2. **排查问题**
   - 查看错误信息和原因
   - 分析处理失败的文件

3. **保存日志**
   - 点击"保存日志"按钮
   - 选择保存位置和文件名
   - 保存为 `.txt` 文件

4. **清空日志**
   - 日志过多时清空
   - 保持界面整洁

## 🎨 界面设计原则

### 1. 功能分离
- 每个页面专注于单一功能
- 避免功能过度集中
- 降低用户认知负担

### 2. 卡片化布局
- 使用 CardWidget 分组相关功能
- 视觉层次清晰
- 便于扩展新功能

### 3. 响应式设计
- 窗口最小尺寸: 900x600
- 推荐尺寸: 1200x800
- 支持窗口缩放

### 4. 一致性
- 统一的按钮样式
- 一致的配色方案
- 统一的交互反馈

### 5. 实时反馈
- 操作立即响应
- 进度实时更新
- 错误及时提示

## 🔧 技术实现

### 页面结构

```python
class FluentMainWindowV2(FluentWindow):
    def __init__(self):
        # 创建四个界面
        self.process_interface = QWidget()      # 文件处理
        self.script_interface = QWidget()       # 脚本管理
        self.settings_interface = QWidget()     # 设置
        self.log_interface = QWidget()          # 日志

        # 添加到导航
        self.addSubInterface(self.process_interface, FluentIcon.HOME, "文件处理")
        self.addSubInterface(self.script_interface, FluentIcon.CODE, "脚本管理")
        self.addSubInterface(self.settings_interface, FluentIcon.SETTING, "设置")
        self.addSubInterface(self.log_interface, FluentIcon.DOCUMENT, "日志")
```

### 线程安全

- **处理线程**: 使用 QThread 避免 UI 卡顿
- **信号槽机制**: 线程间通信
- **UI 更新**: 通过信号在主线程更新 UI

### 配置管理

- **配置文件**: `config/settings.json`
- **自动保存**: 设置修改后自动保存
- **持久化**: 关闭程序后配置不丢失

## ⚠️ 注意事项

1. **Photoshop 必须运行**
   - 处理前确保 Photoshop 已启动
   - COM 接口需要 Photoshop 进程

2. **路径配置**
   - Photoshop 路径必须正确
   - 脚本目录需要包含 .jsx 文件
   - 备份目录需要有写入权限

3. **并发数设置**
   - 不要超过 CPU 核心数
   - 过高可能导致 Photoshop 响应缓慢

4. **日志管理**
   - 长时间运行会积累大量日志
   - 定期清空或保存后删除
