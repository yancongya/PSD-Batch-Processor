# 自动模式使用指南

## 概述

自动模式（Auto Mode）是 PSD Batch Processor 的一项强大功能，允许用户以无人值守的方式批量处理 PSD 文件。当启用自动模式时，所有 JSX 脚本中的确认对话框都会被自动处理，无需人工干预。

## 核心特性

### 1. 自动确认对话框
- **功能**: 自动点击确认按钮，跳过手动确认步骤
- **适用场景**: 批量处理大量文件时，避免逐个确认
- **实现方式**: 通过临时 JSON 文件向 JSX 脚本传递参数

### 2. 详细输出模式
- **功能**: 在日志中显示详细的处理信息
- **适用场景**: 调试脚本或查看详细处理过程
- **输出内容**: 脚本执行的每一步操作

### 3. 批量模式标志
- **功能**: 告知 JSX 脚本当前处于批量处理模式
- **适用场景**: 脚本可以根据模式调整行为

1. **自动模式主开关**
   - 位置: "自动模式 (无人值守)" 复选框

## 使用流程

### 1. 启用自动模式

1. 勾选 **"自动模式 (无人值守)"** 复选框
2. 根据需要配置 **"跳过确认"** 和 **"详细输出"** 选项
3. 点击 **"💾 保存"** 按钮保存配置

### 2. 配置处理任务

1. 选择 Photoshop 路径
2. 选择脚本目录和具体脚本
3. 添加要处理的 PSD 文件或文件夹
4. 配置备份目录和其他选项

### 3. 开始处理

1. 点击 **"▶️ 开始处理"** 按钮
2. 系统会显示确认对话框（如果未启用"跳过确认"）
3. 处理开始后，所有确认对话框都会自动处理
4. 查看日志面板了解处理进度

### 4. 监控处理过程

- **进度条**: 显示总体处理进度
- **日志面板**: 显示详细的处理信息（如果启用详细输出）
- **文件列表**: 显示每个文件的处理状态

## JSX 脚本开发

### 自动模式兼容性

要让 JSX 脚本支持自动模式，需要在脚本开头添加参数读取代码：

```javascript
// ===== 参数读取系统 =====
function readScriptArgs() {
    var defaultArgs = {
        auto_mode: false,
        skip_confirmation: false,
        verbose: false,
        batch_mode: false
    };

    try {
        if (typeof $.ext !== 'undefined' && $.ext.length > 0) {
            var argsFilePath = $.ext[0];
            if (argsFilePath && argsFilePath.length > 0) {
                var file = new File(argsFilePath);
                if (file.exists) {
                    file.open("r");
                    var content = file.read();
                    file.close();
                    var args = eval("(" + content + ")");
                    if (args && typeof args === 'object') {
                        if (args.verbose) {
                            $.writeln("✓ 自动模式已启用");
                        }
                        return args;
                    }
                }
            }
        }
    } catch (e) {
        $.writeln("⚠ 参数读取失败: " + e.message);
    }
    return defaultArgs;
}

// 便捷函数
function autoConfirm(message, defaultResult) {
    var args = readScriptArgs();
    if (args.skip_confirmation || args.auto_mode) {
        if (args.verbose) {
            $.writeln("自动确认: " + message);
        }
        return defaultResult;
    } else {
        return confirm(message);
    }
}
```

### 使用自动确认

将脚本中的 `confirm()` 调用替换为 `autoConfirm()`：

```javascript
// 原代码
if (confirm("确定要处理吗？")) {
    // 处理逻辑
}

// 自动模式兼容代码
if (autoConfirm("确定要处理吗？", true)) {
    // 处理逻辑
}
```

## 配置保存

### 自动模式配置

自动模式配置会保存到全局配置中，包括：
- `enabled`: 自动模式是否启用
- `skip_confirmation`: 是否跳过确认
- `verbose`: 是否详细输出
- `batch_mode`: 批量模式标志

### 配置文件位置

配置保存在内存中，程序重启后会恢复默认值。如需持久化保存，需要在 `app/config/settings.py` 中添加相应字段。

## 注意事项

### 安全性
- 自动模式会跳过所有确认对话框，请确保脚本逻辑正确
- 建议先在少量文件上测试，确认无误后再批量处理
- 备份功能默认启用，请确保备份目录有足够的空间

### 性能
- 自动模式不会显著影响处理速度
- 详细输出模式可能会稍微增加日志量
- 建议在处理大量文件时启用自动模式

### 兼容性
- 自动模式需要 JSX 脚本支持参数读取
- 旧脚本在自动模式下仍会显示确认对话框
- 建议更新旧脚本以支持自动模式

## 故障排除

### 问题: 自动模式不生效
**解决方案**:
1. 检查 JSX 脚本是否包含参数读取代码
2. 确认自动模式开关已启用并保存
3. 查看日志中的详细输出信息

### 问题: 脚本仍然显示确认对话框
**解决方案**:
1. 检查脚本是否使用了 `autoConfirm()` 函数
2. 确认 "跳过确认" 选项已启用
3. 检查参数文件是否正确传递

### 问题: 参数文件读取失败
**解决方案**:
1. 检查临时目录权限
2. 查看日志中的错误信息
3. 确认脚本的参数读取代码正确

## 最佳实践

1. **测试验证**: 先用少量文件测试自动模式
2. **备份重要**: 确保备份功能正常工作
3. **逐步启用**: 先启用"详细输出"，熟悉后再禁用
4. **脚本更新**: 更新旧脚本以支持自动模式
5. **监控进度**: 处理过程中定期检查日志

## 技术实现

### 参数传递机制
1. Python 创建临时 JSON 文件
2. 通过 `$.ext[0]` 传递给 JSX 脚本
3. JSX 脚本读取并解析参数
4. 根据参数决定是否跳过确认

### 配置管理
- 使用 `AutoModeConfig` 类管理配置
- 通过 `save_auto_mode_config()` 保存配置
- 通过 `get_auto_mode_config()` 读取配置

### UI 集成
- 自动模式开关控制相关选项的可用性
- 配置在保存时同步到全局配置
- 启动时从全局配置加载界面状态
